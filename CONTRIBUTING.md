# Contributing to WorldLines

**Language:** [English](./CONTRIBUTING.md) · [简体中文](./CONTRIBUTING.zh.md) · [日本語](./CONTRIBUTING.ja.md) · [한국어](./CONTRIBUTING.ko.md)

> Status: Preview — the contribution flow will evolve with the project.

Thanks for considering a contribution. WorldLines is in public preview; the
contribution flow is intentionally lightweight for v0.1.

## Before you start

- The **engine source** (`neonrp`) is currently closed-preview. This
  repository distributes **installation entry points, documentation, and the
  `stoneford` starter template**. Most community contributions will land here
  or in the [`stoneford-worldlines`](https://github.com/LudicDynamics/stoneford-worldlines)
  submodule, not in the engine binary.

## What we welcome

| Area                         | How to contribute                                |
|------------------------------|--------------------------------------------------|
| Documentation fixes          | PR against these `*.md` files                    |
| Install-path issues          | Issue with OS / uv / Python version              |
| Stoneford content            | PR to the `stoneford` submodule repo             |
| New templates                | RFC issue first, then PR with a working starter  |
| Bug reports against engine   | Issue with repro steps + `neonrp tui --verbose` log |
| LLM provider adapters        | RFC issue — we'll advise engine-side vs shim     |

## Workflow

1. Open an issue describing the problem or proposal.
2. For docs / templates, fork and submit a PR.
3. For engine behavior, wait for a maintainer to confirm scope before
   coding — engine source is not public yet.
4. All contributions require a clear description of *what* changed and
   *why*.

## Style

- Documentation: markdown, ASCII-safe headings, no emoji.
- Command blocks: copy-pasteable, no placeholder shell prompts.
- English primary; Chinese annotations are fine inline when helpful.
- No generated footers (e.g. "Generated by …").

## Testing your changes

For doc-only changes: skim the cross-links locally; optionally run
`markdownlint` if you have it.

For template changes, verify in a scratch directory:

```bash
cd /tmp && mkdir verify && cd verify
neonrp init
neonrp game new --template <your-template>
neonrp tui
```

## Code of conduct

Be decent. Assume good faith. Criticize ideas, not people. A formal CoC
document will be adopted before v0.2.

## License Asymmetry

WorldLines has an intentional split between engine and data:

- **WorldLines engine** (`neonrp`): All rights reserved, proprietary preview.
- **`templates/stoneford-worldlines/`**: AGPL-3.0-only.
- Contributions to `stoneford-worldlines` are licensed under AGPL-3.0.
- Contributions do **not** grant any rights to the engine core.
- Contributors should ensure they can license their work under AGPL-3.0
  before submitting PRs to the stoneford template.
