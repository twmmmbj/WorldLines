# WorldLines へのコントリビュート

**Language:** [English](./CONTRIBUTING.md) · [简体中文](./CONTRIBUTING.zh.md) · [日本語](./CONTRIBUTING.ja.md) · [한국어](./CONTRIBUTING.ko.md)

> ステータス: Preview — コントリビュートフローはプロジェクトと共に進化します。

貢献のご検討ありがとうございます。WorldLines は公開プレビュー中で、v0.1 のコントリビュートフローは意図的に軽量です。

## はじめに

- **エンジンソース**（`neonrp`）は現在クローズドプレビューです。本リポジトリは **インストール入口・ドキュメント・`stoneford` スターターテンプレート** を配布します。コミュニティ貢献の多くはここか [`stoneford-worldlines`](https://github.com/LudicDynamics/stoneford-worldlines) サブモジュールに入り、エンジンバイナリには入りません。

## 歓迎する貢献

| 領域                         | 貢献方法                                           |
|------------------------------|----------------------------------------------------|
| ドキュメント修正             | 本 `*.md` ファイルへ PR                            |
| インストール経路の問題       | OS / uv / Python バージョン付きで issue            |
| Stoneford コンテンツ         | `stoneford` サブモジュールリポジトリへ PR          |
| 新テンプレート               | 先に RFC issue、動作する starter と共に PR         |
| エンジンのバグ報告           | repro 手順と `neonrp tui --verbose` ログ付きで issue |
| LLM プロバイダアダプタ       | まず RFC issue — エンジン側か shim か助言します    |

## ワークフロー

1. 問題または提案を記述した issue を開く。
2. ドキュメント / テンプレートは fork して PR。
3. エンジン挙動はメンテナがスコープ確認するまでコード開始を待つ — エンジンソースはまだ非公開。
4. すべての貢献に *何を* *なぜ* 変えたかの明確な説明が必要。

## スタイル

- ドキュメント: markdown、ASCII セーフな見出し、emoji なし。
- コマンドブロック: コピペ可能、偽のシェルプロンプトなし。
- 主言語は英語、必要な箇所に中国語アノテーションは OK。
- 生成フッター（"Generated by …" 等）禁止。

## 変更のテスト

ドキュメントのみ: クロスリンクを流し読み。`markdownlint` があれば実行。

テンプレート変更は一時ディレクトリで確認:

```bash
cd /tmp && mkdir verify && cd verify
neonrp init
neonrp game new --template <your-template>
neonrp tui
```

## 行動規範

節度を保ち、善意を前提に。アイデアを批判し、人を批判しない。完全な方針は
[CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) を参照し、非公開の脆弱性報告は
[SECURITY.md](SECURITY.md) に従ってください。

## License Asymmetry（ライセンスの非対称性）

WorldLines はエンジンとデータの間で意図的な分割を行っています：

- **WorldLines エンジン**（`neonrp`）: All rights reserved、プロプライエタリなプレビュー。
- **`templates/stoneford-worldlines/`**: AGPL-3.0-only。
- `stoneford-worldlines` へのコントリビュートは AGPL-3.0 の下でライセンスされます。
- コントリビュートはエンジンコアに対するいかなる権利も付与**しません**。
- stoneford テンプレートに PR を提出する前に、コントリビュータは自身の作品を AGPL-3.0 でライセンスできることを確認してください。
