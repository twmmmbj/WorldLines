# 为 WorldLines 贡献

**Language:** [English](./CONTRIBUTING.md) · [简体中文](./CONTRIBUTING.zh.md) · [日本語](./CONTRIBUTING.ja.md) · [한국어](./CONTRIBUTING.ko.md)

> 状态：Preview — 贡献流程会随项目一起演进。

感谢你考虑贡献。WorldLines 处于公开预览期；v0.1 的贡献流程有意保持轻量。

## 开始前

- **引擎源码**（`neonrp`）目前处于闭源预览。本仓库分发 **安装入口、文档、以及 `stoneford` 起步模板**。大部分社区贡献会落在这里或 [`stoneford-worldlines`](https://github.com/LudicDynamics/stoneford-worldlines) 子模块，而不是引擎二进制。

## 我们欢迎的贡献

| 领域                         | 贡献方式                                         |
|------------------------------|--------------------------------------------------|
| 文档修正                     | 对这些 `*.md` 文件提交 PR                        |
| 安装路径问题                 | 开 issue，附带 OS / uv / Python 版本             |
| Stoneford 内容               | 向 `stoneford` 子模块仓库提交 PR                 |
| 新模板                       | 先开 RFC issue，再带可工作的 starter 提交 PR     |
| 针对引擎的 bug 报告          | 开 issue，附 repro 步骤和 `neonrp tui --verbose` 日志 |
| LLM provider 适配器          | 先开 RFC issue —— 我们会建议是做在引擎还是 shim 层 |

## 流程

1. 先开 issue 描述问题或提案。
2. 文档 / 模板：fork 后提交 PR。
3. 涉及引擎行为的变更：先等维护者确认范围再动手，引擎源码尚未公开。
4. 所有贡献都要清楚说明 *改了什么* 与 *为什么改*。

## 风格

- 文档：markdown，ASCII-safe 标题，不用 emoji。
- 命令块：可直接复制粘贴，不要伪造 shell 提示符。
- 主语言英文；必要时内联中文注释没问题。
- 不要生成底部 footer（例如 "Generated by …"）。

## 测试你的改动

仅文档：扫一眼交叉链接；有 `markdownlint` 顺手跑一下也行。

模板改动：在一个临时目录里验证：

```bash
cd /tmp && mkdir verify && cd verify
neonrp init
neonrp game new --template <your-template>
neonrp tui
```

## License Asymmetry（许可非对称性）

WorldLines 在引擎与数据之间刻意做了拆分：

- **WorldLines 引擎**（`neonrp`）：保留所有权利，闭源预览。
- **`templates/stoneford-worldlines/`**：AGPL-3.0-only。
- 对 `stoneford-worldlines` 的贡献按 AGPL-3.0 授权。
- 贡献**不**授予对引擎核心的任何权利。
- 向 stoneford 模板提交 PR 前，贡献者需确保自己有权以 AGPL-3.0 授权其作品。
